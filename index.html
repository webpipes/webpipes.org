What are Webpipes?

Essentially, you can think of webpipes as UNIX programs. They are small utilities available over HTTP that do one thing, and do it well. Webpipes follow the Unix philosophy:

  1. Write programs that do one thing and do it well.
  2. Write programs to work together.
  3. Write programs to handle text streams, because that is a universal interface.

The magic comes when you start chaining pipes together. You can create really amazing workflows by combining a few small pieces together.

How do they work?

Since a Webpipe is essentially just a script that receives input via HTTP POST/GET, there's not much else to know than that.

Where can I find a full list webpipes?

You can view available webpipes on the Webpipes.org GitHub.

Is there a Webpipe specification?

To participate in the wider Webpipe ecosystem, a few conventions have been adopted which enables webpipes to "speak" to one another. Check out the Webpipe Spec.

What's the difference between Webhooks and Webpipes?

Although, webhooks share much in common with webpipes, each solve different problems. Webhooks were designed to solve the push/poll issues. Essentially, a Webhook is a URL that responds to events. For example, Github implements webhooks. On post-receive, Github calls a user-defined URL like a callback. If also feeds along a few arguments that tells you about that commit.

Likewise, Webpipes are essentially simple scripts publicly ascessible via a URL. A webpipe is stateless. You POST/GET some input, and it'll provide you some output.

Why should I care?

Although not yet widely implemented, Webpipes offer developers more leverage. By connecting small and independent units, the whole becomes greater than the sum of its parts.

But the real beauty is the potential for Webpipes to enable a new bred of programs. Some folks have even experimented with wrapping Unix programs around HTTP.

How do I implement Webpipes?

Since Webpipes are basically just scriptlets, they're easy to implement. Read the Webpipes Spec to learn the specifics.
